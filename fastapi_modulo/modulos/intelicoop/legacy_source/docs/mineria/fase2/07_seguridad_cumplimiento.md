# 2.7 Seguridad y Cumplimiento

## Clasificacion de datos
- Publico: catalogos no sensibles.
- Interno: metadatos operativos.
- Sensible: datos personales y financieros de socios.

## Controles minimos
- Acceso por rol con minimo privilegio.
- Cifrado en transito para APIs.
- Enmascaramiento de datos sensibles en reportes no operativos.
- Trazas de acceso a datos sensibles.

## Retencion y respaldo
- Definir retencion de historicos analiticos.
- Respaldos periodicos de tablas criticas.
- Procedimiento de restauracion probado.

## Cumplimiento
- Uso legitimo y minimizacion de datos.
- Eliminacion/anonimizacion de datos cuando aplique normativa interna/externa.
